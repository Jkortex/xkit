# 资源管理

## 存储策略

- **文件存储名**: SHA256(content) + 原始扩展名
- **存储目录**: `{storage_dir}/{sha256[:2]}/{sha256[2:]}.{ext}` 两级哈希目录
- **后端**: 本地文件系统（可扩展 S3）

## 访问控制

- 所有资源按 `owner_user_id` 隔离
- 资源上传后返回稳定 UUID，通过 `GET /resources/:id` 访问
- 访问需要鉴权（Cookie 或 API Key）

## 元数据

| 字段        | 说明                 |
| ----------- | -------------------- |
| `id`        | 存储名（SHA256）     |
| `file_name` | 上传时的原始文件名   |
| `size`      | 文件大小             |
| `mime_type` | 自动检测的 MIME 类型 |

## 垃圾回收

- 孤儿资源 = 既未被 `memo` 表引用、也未被 `memo_history` 表引用的文件
- 与 Memo 解除关联时不会立即删除资源（GC 异步执行）
- 历史版本中的资源引用受保护，防止回滚后资源丢失
