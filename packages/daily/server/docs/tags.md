# 标签体系

## 标签提取（Tag Extraction）

扫描内容的**末尾标签块**，自底向上逐行扫描：

1. 每行若全部由 `#Token` 组成 → 提取为正式标签
2. 遇到非标签行 → 停止
3. 正文中的 `#tag` **不被提取**（仅作纯文本保留）

```
正文内容
#Architecture #Layering    ← 被提取
```

格式要求：`#TagName`（`#` 后紧跟标签名，无空格）。标签名 1-32 字符，支持中文。

## 标签集（Tag Sets）

预定义的筛选条件组合，按**分组**组织：

- `tags_any`: 命中任一即匹配（OR）
- `tags_all`: 全部命中才匹配（AND）
- `tags_exclude`: 排除命中标签（NOT）

### 默认分组

启动时自动创建 3 组默认标签集：

- **工作流**: 待办事项 / 已完成 / 阻塞中
- **阅读管理**: 待阅读 / 正在阅读 / 已读完
- **项目追踪**: 需求 / 缺陷 / 优化

## 标签治理

| 操作     | API                             | 说明                                 |
| -------- | ------------------------------- | ------------------------------------ |
| 重命名   | `POST /tags/rename`             | 目标不存在则重命名，已存在则自动合并 |
| 批量合并 | `POST /tags/merge`              | `sources[] → target`                 |
| 别名管理 | `GET/POST/DELETE /tags/aliases` | 写入/查询路径自动解析为 canonical    |
| 审计     | `GET /tags/audits`              | 治理操作流水记录                     |

所有治理操作均计入审计表 `tag_governance_audit`。
